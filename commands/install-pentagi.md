# /sec-kit install-pentagi

Opt-in installation of the PentAGI autonomous penetration testing platform (by vxcontrol).

## Why this is a separate command

PentAGI is not a skill — it's a full-stack web application:
- Go backend with REST + GraphQL API
- React + TypeScript web UI
- Docker Compose stack (PostgreSQL/pgvector, Neo4j, Grafana, Langfuse, etc.)
- Multi-agent AI system with 20+ built-in security tools
- Deployed via Docker Compose or interactive installer

The kit creates a thin wrapper skill that lets Claude Code interact with PentAGI's API, plus a helper script for deployment management.

## Prerequisites

- Docker and Docker Compose (or Podman)
- Minimum 2 vCPU, 4GB RAM, 20GB disk
- At least one LLM provider API key (OpenAI, Anthropic, Gemini, Ollama, etc.)
- Internet access for pulling images

# PentAGI — Autonomous Penetration Testing Platform

## When to Use

- User asks to run an automated penetration test against a target
- User wants to create, monitor, or retrieve pentest flows via PentAGI
- User needs vulnerability reports generated by AI agents
- User wants to manage the PentAGI deployment (start/stop/status)

## Prerequisites

- PentAGI running at https://localhost:8443 (or configured PENTAGI_URL)
- API token set in PENTAGI_API_TOKEN environment variable
- Docker containers healthy (check with `../.pentagi/scripts/pentagi-status.sh`)

## Deployment Management

```bash
# Start PentAGI
../.pentagi/scripts/pentagi-start.sh

# Stop PentAGI
../.pentagi/scripts/pentagi-stop.sh

# Check status
../.pentagi/scripts/pentagi-status.sh

# View logs
docker compose -f ../.pentagi/docker-compose.yml logs -f
```

## API Access

Base URL: `${PENTAGI_URL:-https://localhost:8443}`
Auth: Bearer token via `PENTAGI_API_TOKEN`

### Create a new pentest flow (REST)

```bash
curl -s -X POST "$PENTAGI_URL/api/flows" \
  -H "Authorization: Bearer $PENTAGI_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Web App Pentest",
    "description": "Full penetration test of target application",
    "target": "https://target-app.example.com",
    "type": "web"
  }'
```

### List flows

```bash
curl -s "$PENTAGI_URL/api/flows" \
  -H "Authorization: Bearer $PENTAGI_API_TOKEN"
```

### Get flow status and results

```bash
curl -s "$PENTAGI_URL/api/flows/<flow-id>" \
  -H "Authorization: Bearer $PENTAGI_API_TOKEN"
```

### GraphQL API

```bash
curl -s -X POST "$PENTAGI_URL/graphql" \
  -H "Authorization: Bearer $PENTAGI_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "{ flows { id name status target } }"
  }'
```

## Workflow

1. Verify PentAGI is running: `../.pentagi/scripts/pentagi-status.sh`
2. Create a flow with the target scope and test type
3. Monitor flow progress via API polling
4. Retrieve vulnerability report when flow completes
5. Present findings to the user with severity ratings and remediation steps

## Safety Notes

- PentAGI performs ACTIVE security testing — ensure you have AUTHORIZATION
- Only test targets you own or have explicit written permission to test
- The Docker sandbox isolates tool execution but network traffic reaches the target
- Review generated reports before sharing — they contain exploit details
```

Write `.pentagi/scripts/pentagi-start.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
COMPOSE_FILE="$SCRIPT_DIR/../docker-compose.yml"
ENV_FILE="$SCRIPT_DIR/../.env"

if [ ! -f "$ENV_FILE" ]; then
  echo "ERROR: .env not found in .pentagi/"
  echo "Copy .env.example to .env and configure your LLM provider keys first:"
  echo "  cp .pentagi/.env.example .pentagi/.env"
  exit 1
fi

echo "Starting PentAGI..."
docker compose -f "$COMPOSE_FILE" --env-file "$ENV_FILE" up -d
echo "PentAGI started. Access at https://localhost:8443"
echo "Default credentials: admin@pentagi.com / admin"
echo ""
echo "Waiting for services to be healthy..."
sleep 5
docker compose -f "$COMPOSE_FILE" ps
```

Write `.pentagi/scripts/pentagi-stop.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
COMPOSE_FILE="$SCRIPT_DIR/../docker-compose.yml"

echo "Stopping PentAGI..."
docker compose -f "$COMPOSE_FILE" down
echo "PentAGI stopped."
```

Write `.pentagi/scripts/pentagi-status.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
COMPOSE_FILE="$SCRIPT_DIR/../docker-compose.yml"

echo "PentAGI Service Status"
echo "======================"
echo ""

if ! docker compose -f "$COMPOSE_FILE" ps --quiet 2>/dev/null | grep -q .; then
  echo "PentAGI is NOT running."
  echo "Start with: .pentagi/scripts/pentagi-start.sh"
  exit 1
fi

docker compose -f "$COMPOSE_FILE" ps
echo ""
echo "Web UI: https://localhost:8443"
echo "Default: admin@pentagi.com / admin"
```

Make scripts executable:

```bash
